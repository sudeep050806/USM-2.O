🚀 Getting Started

Step 1: Configure Email (5 minutes)

  1. Open app/src/main/java/com/example/universitysports/helpers/EmailSender.java
  2. Find these lines (around line 16-19):
    private static final String EMAIL_FROM = "your-email@gmail.com";
    private static final String EMAIL_PASSWORD = "your-app-password";
  3. Replace with your Gmail address
  4. Generate App Password:
    • Go to Google Account → Security
    • Enable 2-Factor Authentication
    • Click "App passwords"
    • Select "Mail" & your device
    • Copy 16-character password
  5. Set app password (no spaces)

Step 2: Open in Android Studio & Run

  1. Open Android Studio (Arctic Fox+ recommended)
  2. File → Open → Select project folder
  3. Wait for Gradle sync (2-5 minutes)
  4. Create/start Android emulator (API 24+)
  5. Click Run button (or Ctrl+R)
  6. App installs & launches automatically

Step 3: Test the App

First, try the Admin account:

Test Student registration:

  6. Enter 6-digit OTP (demo mode: 123456)
  7. Verify → Student Dashboard
  8. Book a ground → Check email for confirmation

🔍 OTP Flow Walkthrough

Step-by-step OTP verification process:

  1. User registers: Enters email & password in SignupActivity
  2. DBHelper.saveOTP(): Generates 6-digit random OTP, stores with 2-minute expiry in database
  3. EmailSender.sendOTPEmail(): Runs in background thread (ExecutorService), sends OTP via Gmail SMTP
  4. OTPVerificationActivity: User enters OTP in 6 auto-focusing text boxes
  5. DBHelper.verifyOTP(): Checks OTP exists, not expired, attempts < 3
  6. Success: Mark OTP as used, set user.is_verified=1, create session, redirect to dashboard
  7. Failure: Increment attempts count, remaining attempts shown in red
  8. Expiry: Countdown timer disables resend after 2 minutes; user must request new OTP
  9. Resend: Generates new OTP, resets timer, clears fields

🔧 Technical Highlights

Thread Management

  • Email sending uses ExecutorService (single thread pool)
  • Database operations run quickly on main thread (local SQLite)
  • Long-running tasks (email, verification) use background threads

Input Validation

  • Email format validation (Patterns.EMAIL_ADDRESS)
  • Phone number validation (10-15 digits)
  • Password strength (min 6, with letters & numbers)
  • Required field checking
  • OTP format (exactly 6 digits)

Data Integrity

  • Foreign key constraints in SQLite (user_id, ground_id)
  • Soft delete for grounds/events (is_active flag)
  • Double-booking prevention via isSlotAvailable()
  • OTP single-use, time-limited validation

⚠️ Security Considerations

For Production Deployment:

  • Password Hashing: Currently plaintext. Implement BCrypt.
  • HTTPS: If adding backend API, use HTTPS only.
  • Email Security: Store email credentials in BuildConfig or secure storage.
  • SQL Injection: Already parameterized queries in DBHelper (safe).
  • Rate Limiting: OTP attempts limited to 3 per account.

📞 Support & Troubleshooting

Common Issues & Solutions

❌ Emails Not Sending
  • Ensure you updated EmailSender.java with real credentials
  • Check internet permission in AndroidManifest
  • For Gmail: Verify App Password (16 chars, no spaces)
  • Check Logcat for detailed error messages
  • Some Gmail accounts block less secure apps - use App Password
❌ OTP Not Verified
  • Check OTP table in DB (should have expiry timestamp)
  • Verify countdown timer is counting down
  • Max 3 attempts enforced - check remaining attempts
  • If expired, resend gives new 2-minute window
❌ Database Errors
  • Clear app data: Settings → Apps → University Sports → Clear Data
  • Or uninstall & reinstall (DB will recreate)
  • Check Logcat for SQLite error details
  • Ensure DBHelper.onCreate() calls insertSampleGrounds()
❌ Build/Gradle Issues
  • File → Invalidate Caches & Restart
  • Ensure Android SDK 35 is installed
  • Check all dependencies are synced
  • Clean → Rebuild Project

🔐 Security Warnings

⚠️ IMPORTANT: Before Production Deployment

  1. Password Hashing: DBHelper.hashPassword() currently returns plaintext. Replace with BCrypt or similar (e.g., BCrypt.hashpw(password, BCrypt.gensalt())).
  2. Email Credentials: Never commit real credentials to git. Use BuildConfig fields or environment variables.
  3. HTTPS: If connecting to remote backend, enforce HTTPS only.
  4. Input Sanitization: Already parameterized SQL queries (prevention), continue avoiding string concatenation.
  5. Rate Limiting: OTP attempts limited per account. Consider IP-based rate limiting for API endpoints.
  6. Sensitive Logs: Remove any Log.d statements that may leak user data before release.
